# L1: The Inference Stack

## Introduction

As a practitioner Inference engineer, understanding the inference stack is crucial for optimizing and deploying large language models (LLMs) efficiently. The inference stack refers to the complete pipeline that transforms user input into model-generated output. This includes preprocessing, model computation, and post-processing steps. In this section, we'll dive deep into the full pipeline, tokenization, forward pass, and the autoregressive loop, providing both theoretical foundations and practical implementation insights.

## Full Pipeline Overview

The LLM inference pipeline can be broken down into the following stages:

1. **Input Reception**: Raw text or multimodal input is received.
2. **Preprocessing**: Input is tokenized and formatted for the model.
3. **Model Execution**: The neural network performs the forward pass.
4. **Generation Loop**: For text generation, an autoregressive loop produces tokens sequentially.
5. **Post-processing**: Generated tokens are decoded back to text and formatted for output.

Here's an ASCII diagram illustrating the high-level pipeline:

```
+-------------------+     +-------------------+     +-------------------+
|   User Input      | --> |  Preprocessing    | --> |  Model Execution  |
|   (Raw Text)      |     |  (Tokenization)   |     |  (Forward Pass)   |
+-------------------+     +-------------------+     +-------------------+
                                                       |
                                                       v
+-------------------+     +-------------------+     +-------------------+
| Generation Loop   | --> | Post-processing   | --> |   Final Output    |
| (Autoregressive)  |     |  (Decoding)       |     |   (Text)          |
+-------------------+     +-------------------+     +-------------------+
```

This pipeline must be optimized for low latency and high throughput in production systems.

## Tokenization

Tokenization is the process of converting raw text into a sequence of tokens that the model can process. Tokens are typically subwords, words, or characters, depending on the tokenizer used.

### Theoretical Background

Modern tokenization often uses subword-based methods like Byte-Pair Encoding (BPE) or WordPiece. BPE, introduced by Sennrich et al. in their 2016 paper "Neural Machine Translation of Rare Words with Subword Units," iteratively merges the most frequent byte pairs to create a vocabulary of subwords. This allows handling of out-of-vocabulary words and provides a good balance between vocabulary size and representation granularity.

For LLMs, tokenization is critical because it directly affects the model's input representation and computational efficiency. The choice of tokenizer can impact both the model's performance and the inference speed.

### Practical Implementation

In practice, tokenizers like those from Hugging Face's `transformers` library are widely used. For example, GPT models use a BPE-based tokenizer trained on large corpora.

```python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("gpt2")
tokens = tokenizer.encode("Hello, world!")
print(tokens)  # [15496, 11, 995, 0]
```

For inference engines like vLLM or TensorRT-LLM, tokenization is often handled separately or integrated into the pipeline. In production, batched tokenization is crucial for throughput.

Reference:
- Paper: Sennrich, R., Haddow, B., & Birch, A. (2016). Neural Machine Translation of Rare Words with Subword Units. arXiv:1508.07909
- Blog: Hugging Face Tokenizers Documentation - https://huggingface.co/docs/transformers/main_classes/tokenizer

## Forward Pass

The forward pass is the computation through the neural network to produce logits for the next token prediction.

### Theoretical Background

For transformer-based LLMs, the forward pass involves:
1. Embedding lookup for input tokens
2. Positional encoding addition
3. Multi-head self-attention layers
4. Feed-forward networks
5. Layer normalization and residual connections

The mathematical foundation is the transformer architecture from Vaswani et al.'s 2017 paper "Attention is All You Need."

### Practical Implementation

In PyTorch or similar frameworks, the forward pass is implemented as a series of matrix multiplications and attention computations. For inference, optimized kernels like FlashAttention are used to accelerate this process.

In vLLM, the forward pass is optimized with CUDA kernels for GPU acceleration.

Reference:
- Paper: Vaswani, A., et al. (2017). Attention is All You Need. arXiv:1706.03762
- Blog: The Illustrated Transformer - http://jalammar.github.io/illustrated-transformer/

## Autoregressive Loop

For text generation, LLMs use an autoregressive approach where each token is predicted based on the previous tokens.

### Theoretical Background

The autoregressive generation follows the chain rule of probability: P(x1, x2, ..., xn) = P(x1) * P(x2|x1) * ... * P(xn|x1,...,x{n-1})

At each step, the model outputs a probability distribution over the vocabulary, and a token is sampled.

### Practical Implementation

The loop continues until a stop condition (e.g., end-of-sequence token, max length) is met. Sampling strategies include greedy, top-k, top-p (nucleus), and temperature-based sampling.

In code:

```python
def generate(model, tokenizer, prompt, max_length=50):
    tokens = tokenizer.encode(prompt)
    for _ in range(max_length):
        logits = model.forward(tokens)
        next_token = sample_from_logits(logits)
        tokens.append(next_token)
        if next_token == tokenizer.eos_token_id:
            break
    return tokenizer.decode(tokens)
```

For efficient inference, this loop is optimized with KV caching and batched processing.

Reference:
- Paper: Radford, A., et al. (2019). Language Models are Few-Shot Learners. arXiv:2005.14165 (GPT-3)
- Blog: Hugging Face Generation Strategies - https://huggingface.co/blog/how-to-generate

The inference stack is a complex interplay of preprocessing, computation, and generation. As practitioners, we must optimize each component for production use, balancing accuracy, latency, and resource usage. Understanding these fundamentals allows for better engineering decisions in deploying LLMs.
