# L3: Prefill, Decode & KV Cache

## Introduction

In LLM inference, the distinction between prefill and decode phases is crucial for optimization. The KV cache is a key memory optimization that stores intermediate attention states. This section explores these concepts in depth, including advanced caching techniques and compression methods.

## Prefill vs Decode

### Prefill Phase

The prefill phase processes the entire input prompt in a single forward pass to compute the initial KV cache.

#### Theoretical Background

For a prompt of length L, prefill computes attention for all positions simultaneously, creating the full KV cache.

#### Practical Implementation

In vLLM, prefill is batched and can be chunked for long prompts.

ASCII Diagram:

```
Prompt Tokens: T1 T2 T3 ... TL
      |
      v
  Forward Pass (all positions)
      |
      v
KV Cache: K1..KL, V1..VL
      |
      v
  Last Token Logits
```

### Decode Phase

The decode phase generates tokens autoregressively, reusing the KV cache.

#### Theoretical Background

Each decode step appends one token, updating the KV cache incrementally.

#### Practical Implementation

Decode is memory-bound due to small batch sizes (often 1).

Reference:
- Blog: Prefill and Decode in LLM Inference - https://www.anyscale.com/blog/continuous-batching-llm-inference

## KV Cache Mechanics

The KV cache stores keys and values from previous positions to avoid recomputation.

### Theoretical Background

For attention, QK^T requires all previous K and V. Cache stores them across steps.

Memory usage: O(num_layers * num_heads * seq_len * d_k)

### Practical Implementation

In PyTorch:

```python
class KVCache:
    def __init__(self, num_layers, num_heads, max_len, d_k):
        self.cache = torch.zeros(num_layers, num_heads, max_len, d_k)

    def update(self, layer, head, pos, k, v):
        self.cache[layer, head, pos] = k, v
```

PagedAttention in vLLM manages cache efficiently.

Reference:
- Paper: Kwon, W., et al. (2023). Efficient Memory Management for Large Language Model Serving with PagedAttention. arXiv:2309.06180

## Advanced Caching Techniques

### Chunked Prefill

For long prompts, process in chunks to fit in memory.

#### Theoretical Background

Divide prompt into chunks, compute attention within and across chunks.

#### Practical Implementation

Used in SGLang for efficient long-context inference.

### Prefix Caching

Cache common prefixes across requests.

#### Theoretical Background

If prompts share prefixes, reuse cached KV states.

#### Practical Implementation

Implemented in vLLM for multi-turn conversations.

### Prompt Caching

Cache entire prompts for repeated queries.

Reference:
- Blog: KV Cache Optimization in vLLM - https://vllm.ai/

## KV Cache Compression

### GQA (Grouped-Query Attention)

As discussed in L2, reduces cache size by grouping queries.

### Token Eviction

Remove less important tokens from cache.

#### Theoretical Background

Use attention scores to evict low-attention tokens.

#### Practical Implementation

H2O: Heavy-Hitter Oracle for eviction.

Reference:
- Paper: Zhang, Z., et al. (2024). H2O: Heavy-Hitter Oracle for Efficient Generative Inference of Large Language Models. arXiv:2306.14048

### StreamingLLM

Enables infinite context by maintaining a sliding window.

#### Theoretical Background

Keep recent tokens and global representations.

#### Practical Implementation

Allows efficient long-context generation.

Reference:
- Paper: Xiao, G., et al. (2023). Efficient Streaming Language Models with Attention Sinks. arXiv:2309.17453

## Conclusion

Optimizing prefill and decode with advanced KV cache techniques is essential for efficient LLM inference. These methods balance memory usage, latency, and quality in production systems.
