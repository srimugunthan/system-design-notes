# L7: FlashAttention, Kernel Fusion & Inference Engines

## Introduction

Optimizing attention computation and kernel execution is crucial for efficient LLM inference. FlashAttention and kernel fusion reduce memory accesses and improve parallelism. Inference engines like vLLM provide high-level optimizations for production deployment.

## FlashAttention

FlashAttention optimizes attention computation by reducing memory IO.

### Theoretical Background

Standard attention: O(N^2) time and space. FlashAttention uses tiling to keep intermediate results in fast memory.

IO-aware: Computes softmax online to avoid storing full attention matrix.

### Practical Implementation

Tiling: Divide Q, K, V into blocks that fit in SRAM.

For each block, compute local attention, accumulate globally.

ASCII Diagram:

```
Q, K, V Matrices
     |
     v
Divide into Tiles (fit in SRAM)
     |
     +-------------------+
     | Tile Q1, K1, V1   |
     | Compute local attn |
     | Accumulate softmax |
     +-------------------+
     |
     v
Global Attention Output
```

Reference:
- Paper: Dao, T., et al. (2022). FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness. arXiv:2205.14135

## Kernel Fusion

Fusing multiple operations into single kernels reduces kernel launch overhead and memory transfers.

### Theoretical Background

Instead of separate kernels for linear, activation, etc., fuse them.

Increases operational intensity, better utilizes GPU resources.

### Practical Implementation

In PyTorch, use `torch.compile` or custom CUDA kernels.

For inference, fused attention + FFN kernels.

Reference:
- Blog: Kernel Fusion in Deep Learning - https://pytorch.org/blog/accelerating-pytorch-with-cuda-graphs/

## PagedAttention

PagedAttention manages KV cache like virtual memory.

### Theoretical Background

Divide KV cache into pages, allocate/deallocate as needed.

Reduces memory fragmentation, enables higher throughput.

### Practical Implementation

In vLLM, uses CUDA memory pools for page management.

Reference:
- Paper: Kwon, W., et al. (2023). Efficient Memory Management for Large Language Model Serving with PagedAttention. arXiv:2309.06180

## Batching Strategies

### Continuous Batching

Dynamically batch requests of different lengths.

#### Theoretical Background

Group requests into batches, process simultaneously when possible.

#### Practical Implementation

vLLM scheduler handles iteration-level batching.

### Dynamic/In-flight Batching

Batch requests as they arrive, without waiting.

Reference:
- Blog: Continuous Batching in vLLM - https://www.anyscale.com/blog/continuous-batching-llm-inference

ASCII Diagram for Continuous Batching:

```
Time -->
Request 1: [seq1]
Request 2: [seq2] [seq2+]
Request 3: [seq3] [seq3+] [seq3++]

Batch 1: [seq1, seq2, seq3]
Batch 2: [seq1+, seq2+, seq3+]
```

## Inference Engines

### vLLM

Open-source engine for efficient LLM serving.

#### Features

- PagedAttention
- Continuous batching
- Speculative decoding
- Multi-GPU support

Reference:
- vLLM Paper: Kwon et al. (2023)

### SGLang

Structured generation with efficient kernels.

#### Features

- RadixAttention for prefix caching
- Chunked prefill
- Backend-agnostic

Reference:
- SGLang: https://github.com/sgl-project/sglang

### TensorRT-LLM

NVIDIA's optimized engine.

#### Features

- TensorRT optimizations
- Multi-head attention fusion
- Quantization support

Reference:
- TensorRT-LLM: https://github.com/NVIDIA/TensorRT-LLM

## Conclusion

These optimizations—FlashAttention, kernel fusion, and advanced engines—enable high-throughput LLM inference. Understanding their internals allows engineers to choose the right tools for specific deployment scenarios.
