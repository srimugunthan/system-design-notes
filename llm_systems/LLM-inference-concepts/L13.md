# L13: Structured Output & Evals

## Introduction

Structured output enables LLMs to generate parseable data like JSON, while evaluations ensure model quality and performance. As inference engineers, we implement logit biasing for format adherence and use comprehensive eval harnesses for benchmarking.

## Structured Output

### JSON Schema

Enforce output format with schemas.

#### Theoretical Background

Define expected structure, constrain generation to match.

#### Practical Implementation

Libraries: Outlines, LMQL, Guidance.

```python
import outlines

model = outlines.models.transformers("model_name")
schema = {"type": "object", "properties": {"name": {"type": "string"}, "age": {"type": "integer"}}}
generator = outlines.generate.json(model, schema)
result = generator("Generate a person")
```

Reference:
- Outlines: https://github.com/outlines-dev/outlines

### Logit Biasing

Modify logits to favor certain tokens.

#### Theoretical Background

Add bias to logits for desired tokens (e.g., JSON delimiters).

#### Practical Implementation

In API: `logit_bias` parameter.

For code: bias towards programming tokens.

Reference:
- OpenAI API Documentation

### Guided Decoding

Constrain generation with grammars.

#### Theoretical Background

Use context-free grammars or regex to guide token selection.

#### Practical Implementation

In Outlines: regex-guided generation.

Reference:
- Paper: Willes, J., et al. (2023). Outlines: Structured Text Generation. arXiv:2304.04628

## Benchmarking Methodology

### Latency Profiling

Measure inference speed.

#### Theoretical Background

Time-to-first-token, tokens/second.

Profile different batch sizes, sequence lengths.

#### Practical Implementation

Tools: PyTorch Profiler, Nsight Systems.

```python
import torch.profiler

with torch.profiler.profile() as prof:
    model.generate(...)
print(prof.key_averages())
```

### Quality Evaluation

Assess output correctness.

#### Theoretical Background

Task-specific metrics: BLEU, ROUGE, F1.

Human evaluation for subjective tasks.

#### Practical Implementation

Automated metrics + human-in-the-loop.

Reference:
