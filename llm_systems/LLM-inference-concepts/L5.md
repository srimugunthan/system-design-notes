# L5: Quantization

## Introduction

Quantization reduces model size and inference latency by representing weights and activations with fewer bits. As inference engineers, we use quantization to fit larger models in memory and accelerate computation. This section covers various quantization techniques from floating-point to low-precision integer formats.

## Floating-Point Formats

### FP16 and BF16

Half-precision floating-point formats reduce memory usage by half.

#### Theoretical Background

FP16: 16-bit float, BF16: 16-bit with same exponent range as FP32 but reduced mantissa.

BF16 is preferred for training stability due to preserved dynamic range.

#### Practical Implementation

In PyTorch: `model.half()` or `model.to(torch.bfloat16)`

Tensor cores support BF16 operations efficiently.

Reference:
- Paper: Wang, N., et al. (2018). Mixed Precision Training. arXiv:1710.03740

## Integer Quantization

### INT8 Quantization

8-bit integer quantization.

#### Theoretical Background

Linear quantization: x_q = round(x / scale) + zero_point

Scale and zero_point calibrated on dataset.

#### Practical Implementation

Post-training quantization (PTQ) or quantization-aware training (QAT).

In Hugging Face: `bitsandbytes` for 8-bit quantization.

Reference:
- Blog: Quantization in Transformers - https://huggingface.co/blog/4bit-transformers

### INT4 Quantization

4-bit quantization for extreme compression.

#### Theoretical Background

Further reduces precision, requires careful calibration to maintain accuracy.

#### Practical Implementation

Used in GPTQ and AWQ for sub-4GB models.

## Advanced Quantization Methods

### GPTQ (GPT Quantization)

One-shot quantization using approximate second-order information.

#### Theoretical Background

GPTQ minimizes quantization error by solving for optimal quantized weights layer-by-layer.

Uses Hessian approximation for importance estimation.

#### Practical Implementation

Implemented in `auto-gptq` library.

```python
from auto_gptq import AutoGPTQForCausalLM

model = AutoGPTQForCausalLM.from_quantized("model_name", quantize_config)
```

Reference:
- Paper: Frantar, E., et al. (2022). GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers. arXiv:2210.17323

### AWQ (Activation-aware Weight Quantization)

Protects salient weights during quantization.

#### Theoretical Background

Identifies important weights based on activation magnitudes, quantizes less important ones more aggressively.

#### Practical Implementation

Better perplexity than GPTQ at same bit-width.

Reference:
- Paper: Lin, J., et al. (2023). AWQ: Activation-aware Weight Quantization for LLM Compression and Acceleration. arXiv:2306.00978

## GGUF Format

GGUF (GPT-Generated Unified Format) for quantized model distribution.

### Theoretical Background

Binary format supporting various quantization types, metadata, and architecture info.

### Practical Implementation

Used by llama.cpp for CPU/GPU inference.

Convert models: `python convert.py model.bin --outtype f16`

Reference:
- GGUF Specification - https://github.com/ggerganov/ggml/blob/master/docs/gguf.md

## Conclusion

Quantization is a trade-off between model size, speed, and accuracy. Choosing the right method depends on hardware constraints and quality requirements. Advanced techniques like GPTQ and AWQ enable running large models on consumer hardware.
