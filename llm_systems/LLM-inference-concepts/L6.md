# L6: Speculative Decoding

## Introduction

Speculative decoding accelerates LLM generation by predicting multiple tokens ahead and verifying them in parallel. This technique reduces the sequential nature of autoregressive decoding. As inference engineers, we implement speculative decoding to improve throughput and reduce latency.

## Basic Concept

Instead of generating one token at a time, speculative decoding generates k candidate tokens and verifies them against the target model.

### Theoretical Background

Draft model proposes tokens, target model verifies in one forward pass.

Acceptance probability based on matching logits.

### Practical Implementation

Reduces latency by factor of k/(1+rejection_rate)

- Paper: Chen, C., et al. (2023). Accelerating Large Language Model Decoding with Speculative Sampling. arXiv:2302.01318

## Draft-Target Approach

Use a smaller draft model to propose tokens, large target model to verify.

### Theoretical Background

Draft model is faster but less accurate. Target model provides ground truth.

### Practical Implementation

In vLLM, implemented with `--speculative-draft-model`

Reference:
- Blog: Speculative Decoding in vLLM - https://vllm.ai/

## N-gram Speculative Decoding

Use n-gram statistics as draft model.

### Theoretical Background

Simple n-gram model predicts next tokens based on recent history.

### Practical Implementation

Fast and model-agnostic, but limited for long contexts.

## Advanced Methods

### EAGLE (Extrapolation Algorithm for Greater Efficiency in LLM)

Uses latent representations for drafting.

#### Theoretical Background

Trains a small model on latent space of target model.

#### Practical Implementation

Better acceptance rates than simple draft models.

Reference:
- Paper: Li, Y., et al. (2024). EAGLE: Extrapolation Algorithm for Efficient LLM Inference. arXiv:2401.15077

### Medusa

Parallel decoding with multiple heads.

#### Theoretical Background

Trains additional heads on target model for parallel prediction.

#### Practical Implementation

Achieves high acceptance rates.

Reference:
- Paper: Cai, T., et al. (2024). Medusa: Simple LLM Inference Acceleration Framework with Multiple Decoding Heads. arXiv:2401.10774

## Acceptance Sampling

The verification step uses rejection sampling based on probability ratios.

### Theoretical Background

For each candidate token, compute acceptance probability min(1, p_target / p_draft)

Sample uniformly; if accepted, use token; else, resample from target.

### Practical Implementation

Ensures correctness while allowing speedup.

Reference:
- Original Speculative Sampling Paper

## Conclusion

Speculative decoding is a powerful technique for accelerating LLM inference. Choosing the right draft mechanism depends on available models and computational resources. In production, it can significantly improve response times for long generations.
